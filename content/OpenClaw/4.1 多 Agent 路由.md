在单 Agent 模式下，所有消息都有同一个 Agent 处理，我们可以使用多 Agent 来处理不同的服务或者不同的消息通道。OpenClaw 的多 Agent 路由就是为此设计的。

## 多 Agent 配置

多 Agent 的核心配置由两部分组成：**Agent 列表**（`agent.list`）和**绑定规则**（`binings`）。

### Agent 列表

Agent 列表定义了系统中存在的 Agent 以及各自的属性。每个 Agent 可以拥有独立的模型配置、技能过滤器、身份信息和沙箱策略。

### 绑定

绑定（binding）是将消息路由到指定 Agent 的规则。
