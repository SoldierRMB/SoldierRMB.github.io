随着对话的进行，会话中的消息会越来越多，对于 LLM 来说，模型的上下文窗口是固定的，当对话历史超过窗口大小时，LLM 会直接拒绝处理，所以必须在调用前就裁剪调多余的内容。OpenClaw 通过**上下文裁剪**和**上下文窗口守卫**来管理这个问题。

## 工具结果裁剪：在 LLM 调键旧工具结果

在 Agent 工作过程中，工具调用的结果往往会占用大量的上下文空间。OpenClaw 目前实现了两套裁剪策略：**软裁剪**（Soft Trim）和**硬清除**（Hard Clear）。

### 裁剪配置

默认配置如下：

- 5 分钟缓存 TTL
- 保护最近 3 轮模型回复
- 上下文 > 30% 时触发软裁剪
- 上下文 > 50% 时触发硬清除
- 可裁剪工具结果至少 50K 字符才执行硬清除
- 软裁剪超过 4K 字符的工具结果才触发软裁剪，并分别保留头部和尾部 1500 字符

### 裁剪触发机制

裁剪以 Pi Agent 的扩展注入到 Agent 运行时中，在每次 LLM 调用前，都会检查是否需要裁剪。
