LLM 本身是没有持久记忆的，记忆系统是为了让 Agent 可以跨会话记住关键信息和检索重要信息。

## 记忆文件布局

### 长期记忆：MEMORY.md

`MEMORY.md` 是用户的长期记忆文件，通常包含爱好、规则和知识。用户可以手动编辑，也可以被 Agent 在对话中写入。

通常是这样的格式：

```
# MEMORY.md - Long-Term Memory

## Preferences

- Favorite fruit: apples.
```

### 日期记忆：memory/YYYY-MM-DD.md

`memory/` 下保存的是会话快照记忆。当用户 `/new` 一个新会话时，OpenClaw 会自动将当前会话的关键内容保存到这里。

## 记忆管理器

### MemoryIndexManager

记忆管理器是记忆系统的核心引擎，主要负责：文件索引、向量嵌入、BM25 全文搜索、混合排序、缓存管理。

### 单例缓存

`MemoryIndexManager` 通过静态方法获取实例，使用全局缓存避免重复创建。

### 文件分块

Markdown 文件被分割成固定大小的分块（Chunk），每个分块独立嵌入向量。

### 文件监视与脏标记

管理器使用 `chokidar` 监视记忆文件的变更，并设置 `dirty` 标记，当 Agent 调用 `memory_search` 时，如果 `dirty` 为 `true`，会先触发增量同步再执行搜索——实现**懒索引（Lazy Indexing）**。
