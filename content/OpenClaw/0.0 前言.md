在 AI Agent 遍地开花的今天，我们习惯于将数据交给云端、将对话交给 SaaS、将隐私交给“用户协议”。但如果有一种方式，能让你拥有一个**完全属于自己**的 AI 助手——它运行在你的设备上，数据存储在你的磁盘上，能力可以由你自由扩展——你会不会想了解它是怎么做到的？

**OpenClaw** 就是这样一个项目。它是一个开源的、本地优先的个人 AI 助手，践行着“你的数据，你做主”的理念。它逐渐从一个玩具，逐步变成一个架构精巧、设计严谨的工程系统。

本系列文档是我在深入阅读 OpenClaw 源码、官方文档后的技术分享。

源码分析基于 OpenClaw v2026.3.8 版本。

## 内容概览

本文档按系统层次组织，共分为以下章节：

### 第一章：概述

- [[1.1 什么是OpenClaw]]：OpenClaw 的定位、“本地优先” 理念、单用户设计哲学、与云端 AI 助手的对比

### 第二章：Gateway

- [[2.1 Gateway 设计]]：Gateway 为何选择 WebSocket 而非 REST，以及幂等键设计如何保证操作的可靠性

### 第三章：会话系统

### 第四章：多 Agent 路由

- [[4.1 多 Agent 路由]]：多 Agent 配置与绑定规则，如何将不同消息路由到不同的 Agent
