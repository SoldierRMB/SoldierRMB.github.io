有一些技能会声明一些外部依赖，但这些依赖在用户的机器上不一定安装了。OpenClaw 的技能安装系统可以让用户或者 Agent 一键安装依赖，同时提供技能状态审计能力。

## 技能安装流程

### 安装入口

安装可以由 CLI 或者 Agent 调用，在安装时会进行以下流程：

1. 加载技能条目
2. 查找安装规格
3. 安全扫描
4. 根据安装类型进行安装（`"node" | "brew" | "go" | "uv" | "download"`）
5. 执行安装命令

### 安全扫描

安全扫描检查技能目录中是否包含可疑的代码模式——例如从未知来源下载并执行文件、访问敏感系统路径等。这对于从 ClawHub 安装的第三方技能尤其重要，因为技能内容最终会被注入到 Agent 的提示词中，恶意技能可能试图操纵 Agent 行为。

## 技能状态管理

### 状态报告

在 OpenClaw 中会对每个技能提供完整的状态审计，会生成每个技能的详细审计报告，主要包含：

- 名称
- 描述
- 来源
- 是否内置
- 是否通过资格检查
- 是否被配置禁用
- 是否被白名单阻止
- 声明的依赖
- 缺失的依赖
- 可用的安装选项
