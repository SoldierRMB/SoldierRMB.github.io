LLM 本身设计是通用的，但他并不精通。想要让 Agent 在一些特定领域变得更加专业，可以为他注入**特定领域知识可操作指南**。OpenClaw 的技能系统（Skills）就是为此而设计：一套可热插拔、自描述、自动发现的 Agent 能力扩展机制。

## 什么是技能：可热插拔的 Agent 能力模块

### 设计理念

一个 Markdown 文件就是一项技能。技能文件（`SKILL.md`）通常包含两部分：

1. **Front Matter**：包含名称、描述、依赖要求等
2. **正文**：Agent 使用该技能的流程、示例、最佳实践等

拿 `weather` 这个 Skill 举例：

```
---
name: weather
description: "Get current weather and forecasts via wttr.in or Open-Meteo. Use when: user asks about weather, temperature, or forecasts for any location. NOT for: historical weather data, severe weather alerts, or detailed meteorological analysis. No API key needed."
homepage: https://wttr.in/:help
metadata: { "openclaw": { "emoji": "🌤️", "requires": { "bins": ["curl"] } } }
---

# Weather Skill

Get current weather conditions and forecasts.

## When to Use

✅ **USE this skill when:**

- "What's the weather?"
- "Will it rain today/tomorrow?"
- "Temperature in [city]"
- "Weather forecast for the week"
- Travel planning weather checks

## When NOT to Use

❌ **DON'T use this skill when:**

- Historical weather data → use weather archives/APIs
- Climate analysis or trends → use specialized data sources
- Hyper-local microclimate data → use local sensors
- Severe weather alerts → check official NWS sources
- Aviation/marine weather → use specialized services (METAR, etc.)

## Location

Always include a city, region, or airport code in weather queries.

...
```

这种设计与 API 不同，API 是提供 API 端点供程序调用；而技能是面向 LLM 的“文档”，通过自然语言描述。

### 技能 vs 工具

在 OpenClaw 中，技能和工具是完全不同的两种概念：

| 维度   | 技能（Skill）| 工具（Tool）|
| ---- | ------------ | ------------------------- |
| 本质   | Markdown 文档   | 可执行函数                     |
| 消费者  | LLM（通过系统提示词）| LLM（通过 function calling 注册）|
| 扩展方式 | 新增 `.md` 文件    | 编写代码并注册                   |

技能相当于告诉 Agent 怎么做，而工具相当于赋予 Agent 能力。

## 技能发现

OpenClaw 支持四种来源的技能，按优先级从低到高排列：

| 类型 | 目录位置 | 来源标识 | 说明 |
|------|---------|---------|------|
| **Extra** | 配置的额外目录 | `openclaw-extra` | 插件或自定义目录提供的技能 |
| **Bundled** | `<package>/skills/` | `openclaw-bundled` | 随 OpenClaw 安装包分发的内置技能 |
| **Managed** | `~/.openclaw/skills/` | `openclaw-managed` | 通过 ClawHub 安装的托管技能 |
| **Workspace** | `<workspace>/skills/` | `openclaw-workspace` | 项目级别的自定义技能 |

优先级体现为：**同名技能，高优先级覆盖低优先级**。如果工作区下有一个自定义的 `weather` 技能，它会覆盖内置的 `weather` 技能。

## 技能加载

### 加载流程

OpenClaw 会在四个来源中加载技能，然后解析 Front Matter，提取其中的元数据。

### 资格过滤

不是所有的技能都会被激活，OpenClaw 会通过多个维度来检查其资格：

1. 配置禁用检查
2. 内置技能白名单检查
3. 操作系统兼容性检查
4. 远程平台合规性检查
5. 无条件激活检查
6. 二进制检查
7. 远程二进制检查
8. 任意远程二进制检查
9. 环境变量检查
10. 配置路径检查

所以安装了 `github` 技能但不安装 `gh`CLI 的技能并不会注册成功。

### 技能快照

因为技能热重载的原因，避免技能冲突以及保证性能，引入了技能快照机制。技能快照是在某一时刻激活的所有技能的静态表示，用于注入到系统提示词中。

### 斜杠命令

Skill 可以拥有斜杠命令，让用户可以直接调用，例如输入 `/weather`，命令会经过处理，只保留 `[a-z0-9_]`，并保证唯一性。命令并不会经过 LLM 处理，会直接调用指定技能。
